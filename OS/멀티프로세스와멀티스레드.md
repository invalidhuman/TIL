# 프로세스(Process)와 스레드(Thread)의 정의 및 차이

운영체제(OS)가 시스템 자원을 효율적으로 관리하기 위해 사용하는 두 가지 실행 단위인 프로세스와 스레드에 대해 정리해보았습니다.

## 1. 프로세스 (Process)

#### 정의

- 실행 중인 프로그램(Program in Execution)
- **하드디스크에 저장된 '프로그램'이라는 정적인 상태가 메모리에 올라와 실행되고 있는 동적인 상태**
- 운영체제로부터 독립된 메모리 영역을 할당받는 작업의 단위이다.

#### 메모리 구조

각 프로세스는 독립된 가상 메모리 공간을 가지며, 크게 4가지 영역으로 나뉜다.

- Code (Text): 실행할 프로그램의 코드(명령어)가 저장되는 영역
- Data: 전역 변수, 정적 변수(static)가 저장되는 영역
- Heap: 런타임에 동적으로 할당되는 데이터가 저장되는 영역
  - Object와 같은 가변적인 데이터 (배열, 함수 등)
- Stack: 함수의 매개변수, 지역 변수, 리턴 주소 등 임시 데이터가 저장되는 영역

## 2. 스레드 (Thread)

#### 정의

- 프로세스 내에서 실행되는 흐름의 단위

- 하나의 프로세스는 최소 하나 이상의 스레드(메인 스레드)를 가진다.

- CPU가 실제로 작업을 수행하는 최소 단위

#### 자원 공유 방식

스레드는 프로세스가 할당받은 자원을 공유하며 실행된다.

- 공유하는 영역: Code, Data, Heap 영역을 같은 프로세스 내의 다른 스레드와 공유한다.

- 독립적인 영역: 각 스레드는 자신만의 Stack 영역과 PC(Program Counter) 레지스터를 가진다.

이유: 스레드가 독립적으로 함수를 호출하고 실행 지점을 기록해야 하기 때문

## 3. 멀티 프로세스 vs 멀티 스레드

- 멀티 프로세스 (Multi-processing)
  하나의 응용 프로그램을 여러 개의 프로세스로 구성하여 각 프로세스가 하나의 작업을 처리하도록 하는 것입니다.

- 장점
  - 안정성이 높다.
  - 하나의 자식 프로세스에 문제가 생겨도 다른 프로세스에 영향을 주지 않는다.

- 단점
  - Context Switching 비용: 프로세스 교체 시 메모리 주소 공간까지 교체해야 하므로 오버헤드가 크다.

자원 낭비: 각 프로세스마다 독립된 메모리를 가지므로 중복되는 자원이 많다.

통신 복잡: 프로세스 간 데이터 공유를 위해 IPC(Inter-Process Communication)라는 복잡한 기술이 필요

멀티 스레드 (Multi-threading)
하나의 프로세스 내에서 여러 스레드를 구성하여 자원을 공유하며 작업을 분담하는 것

장점:

효율성: 메모리 공간과 시스템 자원 소모가 적다.

통신 용량: 힙(Heap) 영역을 공유하므로 별도의 통신 기술 없이 데이터를 주고받기 쉽.

응답성: Context Switching 시 스택 영역만 교체하면 되므로 매우 빠르다.

단점:

안정성 문제: 하나의 스레드에서 오류가 발생해 메모리를 오염시키면 프로세스 전체가 종료될 수 있.

동기화 문제: 여러 스레드가 동시에 공유 자원에 접근할 때 데이터 정합성이 깨지는 문제가 발생할 수 있어, 임계 영역(Critical Section) 관리가 필수적
