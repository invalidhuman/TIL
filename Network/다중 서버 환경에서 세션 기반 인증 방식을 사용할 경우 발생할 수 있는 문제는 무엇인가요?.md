

### 1. 다중 서버 환경에서 세션 기반 인즈

세션은 서버의 로컬 저장소에 저장되기 때문에 서버마다 독립적인 세션 저장소를 가진다.

로드밸런서가 트래픽을 분산하는 환경에서 사용자의 로그인 요청이 서버 A로 라우팅되면 세션 데이터는 서버 A에만 생성되고, 클라이언트는 해당 세션 ID를 쿠키로 전달받는다. 이후 동일한 사용자의 요청이 서버 B로 라우팅되면, 서버 B는 클라이언트가 보낸 세션 ID로 자신의 세션 저장소를 조회하지만 일치하는 데이터가 없으므로 해당 사용자를 미인증 상태로 간주하여 인증 오류를 반환한다.

이것이 **세션 불일치(Session Inconsistency)** 문제다.

답변 예시

> 세션은 서버의 메모리나 브라우저의 로컬 스토리지에 저장되기 때문에, 서버마다 독립적인 세션 저장소를 가집니다. 
> 예를 들어 서버 A와 서버 B가 있고, 로드밸런서가 트래픽을 두 서버에 분산한다고 가정하겠습니다.
> 사용자의 로그인 요청이 서버 A로 라우팅되면 세션 데이터는 서버 A에만 생성되고, 
> 클라이언트는 해당 세션ID를 쿠키로 전달받습니다.
> 이후 동일한 사용자의 요청이 로드밸런서에 의해 서버 B로 라우팅되면, 
> B는 클라이언트가 보낸 세션 ID로 자신의 세션 저장소를 조회하지만 일치하는 데이터가 없으므로 해당 사용자를 인증되지 않는 상태로 간주하여 인증 오류(401에러)를 반환하게 됩니다.



### 2. 해결 방법 3가지

세 방식은 단순 나열이 아니라 앞 방식의 한계를 보완하며 발전해온 흐름이다.

#### 1) Sticky Session

동일한 사용자의 요청을 항상 같은 서버로 라우팅하는 방식. 쿠키 또는 IP를 기준으로 서버를 고정한다.

- **장점**: 구현이 단순
- **단점**:
  - 특정 서버에 트래픽 편중 가능
  - 해당 서버 장애 시 세션 데이터 소실 → 해당 사용자 재로그인 필요
- **참고**: IP 기반 방식은 NAT 환경(여러 사용자가 같은 공인 IP 사용)에서 특정 서버 과부하를 유발할 수 있어 쿠키 기반이 더 일반적

#### 2) 세션 클러스터링 (Session Clustering)

특정 서버에 세션 정보가 생성될 때 다른 모든 서버로 복제하는 방식.

- **장점**: 스티키 세션의 트래픽 편중과 세션 소실 문제 해결
- **단점**:
  - 모든 서버에 중복 저장 → 메모리 비효율
  - 복제 과정의 네트워크 트래픽 증가
  - 복제 지연(Replication Lag) 중 다른 서버로 요청이 가면 이전 상태의 세션을 읽는 **최종 일관성(Eventual Consistency)** 문제 발생 (세션 유실이 아닌 일시적 불일치)

#### 3) 스토리지 분리 (Centralized Session Storage)

세션 저장소를 외부(Redis 등)로 분리하여 모든 서버가 공유하는 방식.

- **장점**: 스티키 세션과 세션 클러스터링의 대부분 문제 해결
- **단점**:
  - 외부 저장소가 단일 장애 지점(Single Point of Failure, SPOF)이 될 수 있음
  - HA(고가용성) 구성으로 SPOF를 보완해도 복제 지연으로 인한 일시적 불일치는 남을 수 있음
  - 외부 스토리지 관리를 위한 추가 리소스 필요

#### 비교 요약

|               | Sticky Session | 세션 클러스터링 | 스토리지 분리 |
| ------------- | -------------- | -------- | ------- |
| 트래픽 편중        | 발생 가능          | 없음       | 없음      |
| 서버 장애 시 세션 소실 | 발생             | 없음       | 없음      |
| 메모리 효율        | 좋음             | 나쁨       | 좋음      |
| 복제 지연 문제      | 없음             | 있음       | 있을 수 있음 |
| 구현 복잡도        | 낮음             | 중간       | 중간      |

---

대안 : 토큰 기반 인증 방식을 사용한다.
인증 방식 자체를 Stateless 한 방식으로 전환할 수도 있다.





- 세션 불일치 문제를 설명할 때는 결과를 구체적으로 말한다: "인증 오류(401 Unauthorized)를 반환하거나 로그인 페이지로 리다이렉트"
- 해결 방법은 단순 나열보다 **스티키 세션 → 클러스터링 → 스토리지 분리** 순으로 발전해온 흐름과 함께 설명하면 이해도를 보여줄 수 있다
