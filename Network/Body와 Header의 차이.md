## 네트워크 통신에서 Body(Payload)와 Header의 차이는 무엇인가요?

Body와 Header의 가장 큰 차이는 정보(데이터)의 역할이 다르다는 것입니다.

Header에는 **데이터의 메타 정보**가 담깁니다.

데이터 자체가 아니라 **데이터 수신자가 데이터를 어떻게 처리해야할 지 지침**을 제공합니다.

예를 들어, Content-Type, Authorization, Cache-Control 과 같은 정보가 포함되고, 각각은 정보의 유형, 인증 버오, 캐시 설정 등의 컨텍스트 정보를 전달합니다.

반면 Body는 전송하려는 실제 데이터를 의미합니다.

HTTP 요청에서 서버로 전달하는 JSON 데이터나 form 데이터가 이에 해당합니다. 일반적으로 헤더에 비해 복잡하고 용량이 큰 데이터를 포함합니다.

즉, Header는 네트워크 통신에서 안내 역할을, Body는 본질적인 데이터를 전달하는 역할을 하고 있습니다. 이 둘의 조화가 효율적인 통신을 가능하게 합니다.

### Header 크기에 제한이 있나요?

HTTP 표준에서는 Header의 명시적인 크기 제한을 정해두진 않고 있지만,

Apache나 Nginx와 같은 웹 서버 단에서는 Header의 크기를 8KB ~ 16KB 사이로 설정해두는 경우가 많습니다.

이 제한값을 초과할 경우, 일반적으로 응답코드 413(Content Too Large)를 응답합니다.
