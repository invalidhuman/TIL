

## 1. 핵심 개념: 인증 방식과 전송 메커니즘은 별개다

혼란의 출발점은 이 둘을 하나로 묶어서 생각하는 것이다.

| 구분 | 정의 | 예시 |
|------|------|------|
| **인증 방식** | 사용자가 누구인지 검증하는 원리 | 세션 기반(Stateful), 토큰 기반(Stateless) |
| **전송 메커니즘** | 증표를 서버-클라이언트 간에 주고받는 수단 | Cookie, Authorization 헤더 |

> "쿠키를 쓰면 세션 방식이다"는 틀린 표현이다. 쿠키는 운송 수단일 뿐이며, 세션 ID를 담을 수도 있고 JWT를 담을 수도 있다.

---

## 2. 인증 방식

### 2-1. 세션 기반 인증 (Session-based Authentication)

- **성격**: Stateful — 서버가 로그인 사용자의 정보를 세션 저장소(메모리 또는 DB)에 보관
- **흐름**:
  1. 로그인 성공 시 서버가 `Session ID` 생성
  2. `Set-Cookie` 응답 헤더로 클라이언트에 전달
  3. 이후 요청마다 클라이언트가 쿠키에 `Session ID`를 실어 전송
  4. 서버는 해당 ID로 저장소를 **조회**하여 사용자 식별 (Key-Value 방식)

### 2-2. 토큰 기반 인증 (Token-based Authentication / JWT)

- **성격**: Stateless — 서버가 상태를 저장하지 않고, 인증 정보를 토큰 자체에 담음
- **흐름**:
  1. 로그인 성공 시 서버가 사용자 정보를 담은 JWT를 생성 및 서명
  2. 쿠키 또는 응답 바디로 클라이언트에 전달
  3. 이후 요청마다 클라이언트가 토큰을 포함하여 전송
  4. 서버는 저장소 조회 없이 토큰의 서명을 **해독**하여 유효성 검증

### 2-3. 본질적인 차이

|           | 세션 기반         | 토큰 기반           |
| --------- | ------------- | --------------- |
| 서버 동작     | 저장소 **조회**    | 서명 **해독**       |
| 서버 상태     | Stateful      | Stateless       |
| 즉시 무효화    | 가능 (저장소에서 삭제) | 어려움 (만료 전까지 유효) |
| 다중 서버 확장성 | 세션 공유 문제 발생   | 문제 없음           |

## 3. 전송 메커니즘

### 3-1. 쿠키 (Cookie)

- **HTTP 헤더 키**: `Cookie:`
- **특징**: 브라우저 자동화
  - 서버가 `Set-Cookie` 헤더를 보내면 브라우저가 자동 저장
  - 이후 요청 시 해당 도메인의 쿠키를 브라우저가 자동으로 `Cookie` 헤더에 포함하여 전송
- **장점**: `HttpOnly` 옵션 설정 시 JavaScript 접근 차단 → XSS 공격에 강함
- **단점**: 브라우저 자동 전송 특성상 CSRF 공격에 취약 → `SameSite` 설정 등 추가 대응 필요

### 3-2. Authorization 헤더

- **HTTP 헤더 형태**: `Authorization: Bearer <Token>`
- **특징**: 수동 관리
  - 프론트엔드 코드가 토큰을 직접 저장 (LocalStorage 등)
  - 요청마다 코드로 헤더에 직접 포함
- **장점**: 브라우저가 자동 전송하지 않으므로 CSRF 공격으로부터 안전
- **단점**: LocalStorage에 저장할 경우 XSS 공격에 취약

### 3-3. 전송 수단 조합 정리

| 인증 방식 | 전송 수단                  | 쿠키에 담기는 것               |
| ----- | ---------------------- | ----------------------- |
| 세션 기반 | 쿠키 (일반적)               | Session ID (값 자체가 아닌 키) |
| 토큰 기반 | 쿠키 또는 Authorization 헤더 | JWT 토큰 전체               |


|      | localStorage + Auth 헤더 | HttpOnly 쿠키    | 메모리 + HttpOnly 쿠키 (하이브리드) |
| ---- | ---------------------- | -------------- | ------------------------- |
| XSS  | 취약                     | 강함             | 강함                        |
| CSRF | 안전                     | 취약             | 강함                        |
| 비고   | 민감한 서비스에는 비권장          | SameSite 설정 필요 | 현재 권장 방식                  |
