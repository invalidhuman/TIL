**CSR**

**서버**

**최초 응답**:

최소한의 빈 HTML 파일과 JavaScript 파일을 클라이언트에 전송

**클라이언트**

1. HTML을 표시
   (화면이 비어 있음)
2. JS 파일을 다운로드/실행합니다.

3. **JS 코드가 백엔드 API를 호출**하여 데이터를 가져옵니다. 4. 데이터를 기반으로 **DOM 요소를 생성/삽입**하여 페이지를 완성합니다.

**SSR**

**서버**

매 요청마다

1. 데이터를 가져온다
2. 완성된 HTML을 생성
3. HTML을 클라이언트에 전송

**클라이언트**

1. 완성된 HTML을 즉시 표시

2. JS 파일을 다운로드/실행(Hydration)

3. 이후 상호작용이 가능해짐

**SSG**

**서버**

최초 응답:

빌드 시 미리 생성된 완성된 HTML 파일을 CDN을 통해 클라이언트에 즉시 전송

**클라이언트**

1. 완성된 HTML을 즉시 표시
2. JS 파일을 다운로드/실행합니다. (Hydration)
3. 이후 상호작용이 가능해집니다.

**ISR**

**서버**

**최초 응답:**

SSG와 동일하게 **미리 생성된 HTML**을 전송

**클라이언트**

1. 완성된 HTML을 즉시 표시합니다. 2. JS 파일을 다운로드/실행합니다. (Hydration) 3. 이후 상호작용이 가능해집니다.

**서버의 추가 역할
재검증 시**: 설정된 시간(revalidate)이 경과한 후 요청이 들어오면, **백그라운드에서 데이터를 가져와** 새로운 HTML을 생성하고 캐시를 업데이트합니다.
