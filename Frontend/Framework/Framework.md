# Next.js 도입 배경과 장단점 정리

### 1. Next.js를 사용하는 이유 (장점)

꼭 Next.js가 아니어도, Spring Boot, Angular 등에 해당하는 개념으로 CoC가 있다.

#### CoC (Convention Over Configuration)

개발자가 일일이 세부 사항을 설정하는 대신,
프레임워크가 정한 상식 적인 Convention을 따르면 자동을 동작하게 만드는 방식을 말한다.

React.js와 Next.js 사이에도 이런 관계가 있다.
React를 이용해서 웹 애플리케이션을 구현하기 위해서는 번들러 설정, 라우팅 설정, 다양한 렌더링 방식을 위한 추가 세팅 등 복잡한 과정을 거쳐야 한다.

이러한 작업이 결국 Configuration이며, 시간과 노력이 많이 든다.
많은 개발자들에게 진입 장벽이 되기도 한다.

하지만 Next.js는 이러한 복잡한 과정을 생략하고, 기본적으로 설정된 환경에서 편하게 웹 애플리케이션을 개발할 수 있도록 도와준다.

ex. Next.js의 파일 라우팅 시스템 (디렉터리 구조만으로 페이지를 생성)
Next.js 프로젝트에서 pages.about.js를 만들면 자동으로 /about 경로가 생김.
React에서는 따로 라우팅 설정을 해줘야함
